# Prompt 4g — Root health-check master (X:\CHECKS\)

**Owner:** Codex
**Risk:** Low (only file creation + read-only probes)
**Depends on:** 4a complete (per-NLP `health_check.bat` files must exist first)
**Blocks:** Nothing — but unblocks routine "is the brain healthy" checks

---

## Goal

Stand up `X:\CHECKS\` as the single click-button for **"is the entire brain healthy?"**. Cycles every workflow's `health_check.bat`, aggregates results into one report, returns a clear PASS/FAIL.

This is the second half of David's request: per-NLP health checks (4a Stage 4) PLUS a root-level master that calls them all (4g).

---

## Pre-conditions

- Phase 4a Stage 4 complete: every active workflow folder has `health_check.bat`
- `X:\THEOPHYSICS_PRIMER.md` exists (used in health-check meta-validation)
- `X:\FOLDER_CONVENTIONS.md` is the canonical convention spec

## Required outcome

### Stage 1 — Create `X:\CHECKS\` directory + structure

```
X:\CHECKS\
  README.md                    ← Layer 1 fields, what this is
  RUN_ALL.bat                  ← orchestrator entry point
  run_all.ps1                  ← actual implementation (PowerShell)
  health_report.md             ← LATEST result (overwritten each run)
  history\                     ← timestamped past reports
    health_report_YYYYMMDD_HHmm.md
```

### Stage 2 — `run_all.ps1` implementation

The script must:

1. **Discover** every workflow with a `health_check.bat` under:
   - `X:\<nlp>\health_check.bat` (current root-resident NLPs)
   - `X:\00_WORKFLOWS\<nlp>\health_check.bat` (post-4b structure)
2. **Run each in sequence**, capture stdout + exit code, timeout 60s per check
3. **Validate framework anchors**:
   - `X:\THEOPHYSICS_PRIMER.md` exists, has non-empty `[CANON SLOT]` markers logged (warn if any unfilled)
   - `X:\FOLDER_CONVENTIONS.md` exists
   - `X:\ARCHITECTURE.md` exists
   - Each active workflow has `_AGENT_BRIEF.md`, `README.md`, `RUN.bat`, `config.json`, `00_DROP/`, `OUTPUT/`, `ARCHIVE/`
4. **Probe shared services**:
   - Ollama @ localhost:11434 — `GET /api/tags` returns 200
   - Infinity @ 192.168.1.177:7997 — `GET /health` returns 200
   - Qdrant @ 192.168.1.177:6333 — `GET /` returns 200
   - PostgreSQL @ 192.168.1.177:2665 — connection succeeds (use `pg_isready` if available, else TCP probe)
5. **Aggregate** into `health_report.md`:

```markdown
# Brain Health Report — YYYY-MM-DD HH:MM

**Overall:** PASS | FAIL | DEGRADED
**Checks run:** N total · P passed · F failed · W warnings

## Per-workflow

| Workflow | Status | Duration | Notes |
|---|---|---|---|
| paper-proof-grader | PASS | 1.2s | |
| knowledge-refinery | FAIL | 0.4s | DeBERTa endpoint unreachable |
| ... | | | |

## Shared services

| Service | Status | Latency |
|---|---|---|
| Ollama localhost:11434 | PASS | 12ms |
| Infinity 192.168.1.177:7997 | PASS | 28ms |
| Qdrant 192.168.1.177:6333 | PASS | 19ms |
| Postgres 192.168.1.177:2665 | PASS | 15ms |

## Framework anchors

- THEOPHYSICS_PRIMER.md: present · 4 canon slots unfilled (warning)
- FOLDER_CONVENTIONS.md: present
- ARCHITECTURE.md: present

## Failures (if any)

- stdout/stderr:
  ```

  ```

```

6. **Copy** the report to `history\health_report_<timestamp>.md` after writing the latest

7. **Return exit code:**
   - 0 if all checks PASS
   - 1 if any FAIL
   - 2 if WARNINGS only

## Acceptance check

```powershell
# 1. Structure exists
Test-Path 'X:\CHECKS\RUN_ALL.bat'        # True
Test-Path 'X:\CHECKS\run_all.ps1'        # True
Test-Path 'X:\CHECKS\README.md'          # True
Test-Path 'X:\CHECKS\history'            # True

# 2. Running it produces a report
cd X:\CHECKS
.\RUN_ALL.bat
Test-Path 'X:\CHECKS\health_report.md'   # True
$report = Get-Content 'X:\CHECKS\health_report.md' -Raw
$report -match '^# Brain Health Report'  # True
$report -match 'Overall:\s+(PASS|FAIL|DEGRADED)'  # True

# 3. Per-workflow rows appear
$nlps = 'axioms','knowledge-refinery','paper-proof-grader','link-pull-drop','session-handoff-drop'
$nlps | ForEach-Object { Write-Host "$_ in report? $($report -match $_)" }

# 4. History got a timestamped copy
(Get-ChildItem 'X:\CHECKS\history' -Filter 'health_report_*.md').Count -ge 1  # True
```

All blocks should pass.

## Absolute rules

- **Read-only probes only.** Health check must not write to any workflow's state.
- **No deletes** in `history\`. Rotation/pruning is a separate later prompt.
- **Timeout each per-NLP check at 60s.** Hung NLP shouldn't hang the master.
- **No service mutation.** If Ollama is down, report it — don't try to start it.
- **Use Filesystem MCP for NAS paths if any are introduced.**
