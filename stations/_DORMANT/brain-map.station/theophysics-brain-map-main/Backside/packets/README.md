# Backside Packets

**What this is:** Packet folders are portable work units.
**Owner:** shared
**Status:** scaffold
**Last updated:** 2026-05-16

Packets are where actual work accumulates as it moves through stations.

Keep the packet visible as the unit, but keep its layers separate:

- `PUBLIC/` for reader-facing HTML/Markdown/export material
- `WORKING/` for drafts, notes, station outputs, and remnants
- `MACHINE/` for metadata, provenance, tags, station logs, and indexing hints
- `ARCHIVE/` for superseded, raw, or failed-run artifacts

Do not create a top-level folder for every remnant. Remnants stay inside the packet that produced them.

See `PACKETS_CONVENTION.md` at repo root for the full contract.
