SECTION 3 - RELATIONSHIP TO THEOPHYSICS

Do not use vague labels like "related" unless nothing stronger is justified.

Use one primary relation:

- represent-us: this paper directly represents a claim we make or need.
- near-us: this paper has the same structure but does not explicitly support us.
- cite: this paper should be cited for a narrow claim.
- contrast: this paper is useful because it shows the opposing frame.
- challenge: this paper may break or weaken a claim.
- background: useful context, not load-bearing.
- reject: not useful, too weak, or misleading.

SECTION 5 - 7QS GATE

This is the missing layer that should sit between "paper says something interesting" and "this belongs in the axiom system."

No paper goes directly into the axiom chain. It must pass through 7QS first.

7QS snapshot:

Q0 - Posture
Are we reading humbly and truthfully, or forcing the source to agree with us?

Q1 - Identity
What is it? Fix the object's name, type, scope, stability, source, and math object if present.

Q2 - Location / Domain
Where does it live? Physics, theology, consciousness, information, mathematics, history, or multiple domains?

Q3 - Assertion
What exactly is being claimed? Make it precise enough to kill.

Q4 - Evidence
What supports it? Classify evidence type, strength, independence, and experimental or formal support.

Q5 - Dependencies
What must already be true? Name upstream axioms, laws, definitions, assumptions, and boundary conditions.

Q6 - Consequences
If true, what follows? Map downstream implications, predictions, forced links, and cascade effects.

Q7 - Falsification
How does it die? Name kill conditions, collapse scope, and what would fall if it fails.

- pass
- partial
- fail
- hold for review

SECTION 6 - DERIVED / EXTENDED / CLOSURE LINKS

Core / Derived / Extended / Closure are not four copies of the same content. They are routing levels.

Use them like this:

Core:
Use only when the source touches the floor: distinction, substrate, existence, relation, observation, three-observer closure, infinite source.

Derived:
Use when the source supports a theorem or boundary consequence that follows from the floor.
Examples: information primacy, coherence cannot self-increase, grace external, information conservation, voluntary coupling.

Extended:
Use when the source touches framework-level claims.
Examples: Terminal Observer, Grace Operator, Master Equation, worldview filter, law mapping.

Closure:
Use only for system-level claims about consistency, completeness, independence, Logos theorem, unique solution, or Omega.

Most papers should not touch Closure. If every paper gets Closure language, the layer is being abused.

SECTION 8 - FALSIFICATION / RED FLAGS

Every snapshot must name what could go wrong.

Red flag types:

- source is only analogical
- source supports a weaker claim than we want
- source contradicts a core axiom
- source is outdated
- source has no empirical support
- source is useful but not citable
- source is strong but belongs in a different layer

Falsification note:

If this source is wrong, what breaks?

If our mapping of this source is wrong, what breaks?

PROMPT FOR AI PARTNERS

When processing a paper for the axiom workflow:

1. Do not promote directly from paper to axiom.
2. Build a snapshot first.
3. Run the 7QS gate.
4. Map only exact source claims to exact axiom/layer touchpoints.
5. Preserve uncertainty.
6. If the paper is merely similar, label it near-us, not represent-us.
7. If a citation is recommended, name the exact claim it supports.
8. If the source challenges us, say so directly.

Output only the snapshot unless David asks for the full analysis.
