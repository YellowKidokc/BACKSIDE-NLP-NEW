# 06_Publication — Polished Output

## Purpose
This is your **storefront** — finished work ready for public consumption.

## What Goes Here
- **Papers** — formal academic papers (The Logos Papers series)
- **Essays** — Substack articles, blog posts
- **Book Chapters** — sections of larger works
- **Presentations** — conference talks, lectures
- **Tutorials** — explainer content

## The Publication Standard
A work is publication-ready when:
- ✅ All claims sourced to atoms/molecules/laws
- ✅ Clear structure (intro → body → conclusion)
- ✅ Testable predictions defined
- ✅ Falsification criteria stated
- ✅ Professional formatting
- ✅ Citations complete
- ✅ Peer-reviewed (if academic)
- ✅ SEO optimized (if Substack)

## Types of Publications

### 1. Academic Papers
- Formal structure (abstract, methods, results, discussion)
- Peer review ready
- Suitable for arXiv, journals
- High evidential standard

### 2. Substack Articles
- Accessible language
- Narrative structure
- SEO keywords
- Visual aids
- Premium vs. free designation

### 3. Book Chapters
- Extended development of ideas
- Cohesive narrative arc
- Deep dives

### 4. Presentations
- Slide deck format
- Visual emphasis
- Condensed key points

## Workflow
1. Identify a cluster of related atoms/molecules/laws
2. Create publication note
3. Use template: `Ctrl+T` → "06_Publication_Template"
4. Write draft (pull from existing notes)
5. Link all claims to foundation
6. Add citations
7. Generate abstract (AI-assisted)
8. Format equations
9. Get peer review feedback
10. Finalize and publish

## Publication Routing
Use YAML to control where it goes:
```yaml
publish_to:
  production: true      # Live website
  research: false       # Private vault
  substack: true        # Substack blog
  arxiv: true           # Academic preprint
  journal: false        # Journal submission
```

## Visibility Levels
- `public` — free for everyone
- `premium` — Substack premium tier
- `academia` — journal/conference only
- `private` — internal use only

## Status Progression
- `draft` → initial writing
- `review` → peer feedback stage
- `final` → approved for publication
- `published` → live and public

## Key Outputs
When published, update:
- Related atoms/molecules/laws → add to `demonstrated_in_papers:` field
- Citation metadata
- DOI (if applicable)
- Permalink

## Key Principle
**This is the fruit of all prior work.**

Every published piece should trace back to atoms → molecules → laws. If you can't do that, it's not ready.
