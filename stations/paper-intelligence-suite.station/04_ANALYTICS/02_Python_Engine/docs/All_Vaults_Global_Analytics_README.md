# Global Analytics Engine

This folder contains a standardized, organized system for running both lightweight "native" vault analytics and powerful, deep-dive Python analytics.

## 🧠 Philosophy

This structure provides two complementary analytics workflows:

1.  **Native Tools (`01_Native_Tools`)**: For live, simple, and visual analysis directly within Obsidian using Dataview and Canvas.
2.  **Python Engine (`02_Python_Engine`)**: For complex, data-intensive tasks like content analysis, external API integration, and AI-powered research that are beyond the scope of native tools.

## 📂 New Folder Structure

-   **`01_Native_Tools/`**: Houses pre-built Dataview dashboards and Canvases that can be copied into any project for instant local analytics.

-   **`02_Python_Engine/`**: The core of the deep analytics system.
    -   `data/`: Contains all raw and processed data used by the Python scripts.
    -   `reports/`: Stores the human-readable output and reports generated by the engine.
    -   `scripts/`: Holds all Python scripts for analysis, automation, and visualization.

-   **`03_Dashboards/`**: High-level dashboards that combine native queries with links to the powerful reports generated by the Python engine.

-   **`04_Custom_Modules/`**: Contains specialized, domain-specific Python modules. This is where the "secret sauce" of the analytics engine resides.
    -   `theophysics_analytics/`: The core module for Theophysics-related analysis.

-   **`ARCHIVE/`**: A consolidated archive of all old, unsorted, and legacy files from the previous structure. Nothing has been deleted; it is all here for your review.

## 🚀 How to Use

1.  **Place Python scripts** into the `02_Python_Engine/scripts/` directory.
2.  **Add raw data** (CSVs, JSONs, etc.) to the `02_Python_Engine/data/raw/` directory.
3.  **Run your Python scripts** to perform analysis and generate reports in the `02_Python_Engine/reports/` directory.
4.  **Use the dashboards** in `03_Dashboards/` to get a high-level overview of your vault's analytics.
