# Analytics Scripts & Workflow

This folder contains the scripts that power the Data Analytics Engine. It's designed to be a flexible system that uses the best tool for the job.

## The Workflow Architecture

Our system uses a simple but powerful four-part architecture:

1.  **Dashboard (.md in `01_Dashboards`)**: The user interface. It uses DataviewJS to display data from a JSON file.
2.  **Trigger (.bat)**: A simple batch script that executes a Python script. This acts as the "button" you can press to start an analysis.
3.  **Engine (.py)**: A Python script that performs the heavy lifting. It runs complex analysis that would be too slow or impossible for Dataview.
4.  **Data File (.json)**: The Python script writes its results to a JSON file. The Dashboard reads this file.

This creates a one-way data flow: `Batch -> Python -> JSON -> DataviewJS`.

### Why Use a Batch Script?

Obsidian and Dataview have security limitations and cannot run programs like Python directly. The batch script is a necessary "glue" that allows a user in Obsidian to easily trigger a powerful external script.

## When to Use Dataview vs. Python

Deciding which tool to use is the key to this system.

### Use `Dataview` for:

*   **Querying existing metadata:** Finding all notes with a certain tag (`#concept`), files in a folder, or notes with specific frontmatter.
*   **Simple data aggregation:** Counting files, creating tables of existing data, grouping notes by a property.
*   **Displaying data:** Rendering lists, tables, and task lists from queries.

In short, use Dataview for tasks that are **fast** and involve **reading and displaying** data that is already in your vault.

### Use `Python` for:

*   **Heavy computation:** Any analysis that involves complex calculations, loops over thousands of items, or requires significant processing power.
*   **Complex text analysis:** Natural Language Processing (NLP), sentiment analysis, topic modeling.
*   **Accessing external resources:** Calling APIs, connecting to external databases, accessing the internet.
*   **Modifying files:** While Dataview can't change files, a Python script can be written to programmatically add tags, update frontmatter, etc. (Use with caution!).

In short, use Python for tasks that are **slow, complex, or involve external data or file modifications**.

## Advanced Workflow: Dataview as a Pre-processor

You can combine these tools in a more advanced way:

1.  Create a Dataview query in a note that generates a list of files based on some criteria (e.g., all files tagged `#paper` modified in the last 7 days).
2.  Your Python script can be designed to **read this note first**, parse the file list generated by Dataview, and then run its complex analysis on only those specific files.

This is an extremely powerful way to interactively select a dataset inside Obsidian and then process it with Python.
