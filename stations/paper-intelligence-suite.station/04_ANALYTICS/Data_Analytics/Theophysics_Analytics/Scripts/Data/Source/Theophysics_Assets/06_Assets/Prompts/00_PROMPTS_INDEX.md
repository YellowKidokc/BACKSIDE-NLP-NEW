# Logos Papers: Workflow Prompts Index

## Overview

This directory contains comprehensive prompts for maintaining, improving, and validating the Logos Papers series. Each prompt provides systematic instructions for a specific workflow.

## Purpose

These prompts enable:
- **Consistency** across all 12 papers
- **Quality control** through systematic checking
- **Efficiency** via reusable workflows
- **Onboarding** new AI assistants or human editors
- **Standardization** of complex processes

## Available Prompts

## How to Use These Prompts

### For AI Assistants

**Method 1: Direct Copy-Paste**
1. Read the relevant prompt file
2. Copy the entire prompt
3. Paste into conversation with AI assistant
4. AI assistant follows the systematic instructions

**Method 2: Reference by Name**
"Please follow the procedures in `02_Math_Checker_Validator.md` to check the equations in Paper 7."

**Method 3: Hybrid Approach**
"Using the YAML template from `01_YAML_Header_Generator.md`, create a header for Paper 5 with these specific tags: [list]"

### For Human Editors

**Use as Checklists**:
- Print out relevant prompt
- Check off items as you complete them
- Note issues in margins
- Track completion progress

**Use as Training Materials**:
- Onboard new team members
- Standard operating procedures
- Quality control guidelines
- Style guide reference

### For Automated Tools

**Script Integration**:
- Parse prompts to extract checklists
- Automate validatable checks (e.g., YAML syntax)
- Generate reports based on prompt criteria
- Flag human review needed items

### Versioning

Format: `[Prompt Name] v[Major].[Minor] - [Date]`

- **Minor version**: Clarifications, examples, small additions

## Best Practices

### For Systematic Work
- Use prompts in sequence (see Workflow Sequences)
- Check off items as completed
- Document any deviations from prompt
- Keep notes on improvements for next time

### For Efficiency
- Batch similar tasks (all YAML updates at once)
- Use prompts as templates, customize as needed
- Save completed checklists for reference
- Build on previous work

### For Quality
- Don't skip steps in prompts
- When unsure, err on side of thoroughness
- Multiple passes catch more errors
- Fresh review after time away

### For Collaboration
- Share which prompt you're following
- Note where you deviated and why
- Pass completed checklists to next person
- Update prompts based on team feedback

## Troubleshooting

### "Prompt is too detailed"
- Focus on relevant sections for your task
- Use quick reference for high-priority items
- Build up to full prompt over time

### "Instructions conflict with paper needs"
- Prompts are guidelines, not rigid rules
- Document why you deviated
- Consider updating prompt if deviation makes sense

### "Unsure which prompt to use"
- See Quick Reference table above
- Start with 07 (Final Quality Review) for comprehensive assessment
- When in doubt, use multiple prompts

### "Prompt seems outdated"
- Check for newer version in folder
- Note issues and suggest updates
- Use best judgment for current best practices
