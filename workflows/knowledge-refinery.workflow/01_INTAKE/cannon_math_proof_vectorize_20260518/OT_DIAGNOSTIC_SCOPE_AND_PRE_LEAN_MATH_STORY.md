# OT Diagnostic Scope + Pre-Lean Math Story

Run context: 2026-05-18  
Purpose: classify the current paper as Old Testament diagnostic / pre-cure structure, separate NT preview from NT derivation, and outline the mathematical formalism before Lean 4 submission.

## Scope Classification

This draft should be classified as:

> Old Testament diagnostic structure: the world before the final cure, where law, sacrifice, judgment, covenant, and containment reveal the problem without yet completing the solution.

The paper has touched New Testament concepts, but mostly as endpoint previews:

- mediator
- cross
- resurrection
- distributed Spirit / sanctification
- final restoration

Those should not be treated as fully derived NT modules yet.

## Scope Rule

The OT paper may point toward the NT solution, but it should not solve the NT solution in full.

Good OT ending:

> The system has diagnosed the need for a mediator who can preserve justice, enact mercy, transform the person, and defeat death.

Bad OT ending:

> The full cross/resurrection/sanctification doctrine has now been derived.

The current paper should stop at necessity/shape, not completion.

## What Belongs In The OT Diagnostic Paper

1. Truth and coherence are prior to error.
2. Creation is good and relational.
3. Free will makes rejection possible.
4. Human failure is structurally universal.
5. Perfect holiness creates a threshold, not a human gradient.
6. Justice and mercy cannot be separated without breaking the system.
7. Law reveals and restrains but does not regenerate.
8. Sacrifice covers, costs, and points forward but does not complete.
9. OT severity functions as containment before cure.
10. The unresolved structure requires an external mediator.

## What Should Be Deferred To The NT Paper

1. Incarnation as exact mediator mechanism.
2. Cross as complete atonement event.
3. Resurrection as death-boundary breach.
4. Pentecost / distributed Spirit as internal transformation layer.
5. Sanctification as post-cure coherence dynamics.
6. Church/body as distributed coherence organism.
7. New creation / eschaton as `C -> 1`.

## Pre-Lean Mathematical Story

The current formal story can be expressed before Lean as a staged dynamical model:

```text
dC/dt = O·G(1-C) - S·C
```

Where:

- `C` = coherence / Christological completion variable in later NT layer.
- `S` = sin/decoherence pressure.
- `G` = grace/external restoration operator.
- `O` = openness / voluntary coupling / receptive posture.
- `(1-C)` = distance from full coherence, giving restoration room to operate.

For the OT diagnostic paper, do not overuse the full NT meaning of `C`. Use `C` first as coherence, and only later mark the Christological identity as an NT completion claim.

## OT Phase Math

### Phase 1 — Creation

```text
C ≈ 1, S = 0 or inactive
```

The system begins coherent. Error is not primitive. Corruption requires prior order.

Gap to fill before Lean:

- Define whether initial `C = 1` is literal, idealized, or local to Edenic conditions.

### Phase 2 — Fall

```text
S > 0
dC/dt includes -S·C
```

Sin does not create a rival world; it degrades an already good one. The higher the coherence, the more meaningful the loss.

Gap to fill before Lean:

- Define `S` as an operator, scalar, or family of decoherence actions.
- State whether `S·C` means proportional degradation or category-level rupture.

### Phase 3 — Human Failure / Sign Invariance

```text
self-generated repair cannot flip corrupted orientation
```

The system cannot save itself by using only operations generated from its corrupted state.

Gap to fill before Lean:

- Name the orientation/sign variable.
- Define what counts as a self-generated operation.
- Define the counterexample that would break the theorem.

### Phase 4 — Holiness Threshold

```text
human gradient ≠ divine threshold
```

Humans rightly distinguish sins by harm, intent, and consequence. But before perfect holiness, all sin shares one catastrophic property: unmediated incompatibility.

Suggested formal shape:

```text
H(x) = 1 if x is compatible with unmediated holiness
H(x) = 0 if x contains any unmediated sin-condition
```

Gap to fill before Lean:

- Decide whether this is binary threshold logic or limit behavior.
- Preserve the boundary: sins are not equal in temporal consequence.

### Phase 5 — Justice/Mercy Constraint

Justice alone:

```text
J = 1, M = 0 -> standard preserved, people lost
```

Mercy alone:

```text
J = 0, M = 1 -> people admitted unchanged, destination corrupted
```

Required solution:

```text
J = 1, M = 1, T = 1
```

Where `T` is transformation, not mere relocation.

Gap to fill before Lean:

- Define cost conservation without crude transactionality.
- Decide whether `J`, `M`, and `T` are booleans, constraints, or objective functions.

### Phase 6 — Law As Diagnostic

```text
Law exposes error: E_detect ↑
Law does not internally supply G sufficient for sign reversal
```

Law is holy and good; the defect is not in the law but in the human condition.

Gap to fill before Lean:

- Define diagnosis vs cure.
- Avoid formalizing law as negative.

### Phase 7 — Sacrifice As Provisional Covering

```text
covering ≠ completion
pointer ≠ final payment
```

Sacrifice is covenantally real, but not final-form completion.

Gap to fill before Lean:

- Define covering as temporary state, not fake state.
- Define why animal sacrifice cannot permanently perfect conscience without importing the full NT solution too early.

### Phase 8 — OT Triage / Containment

```text
G available, O limited, transformation not yet distributed
-S·C often dominates
```

This predicts a world of restraint, judgment, covenant repair, sacrifice, exile, prophetic warning, and partial restoration.

Gap to fill before Lean:

- Case-by-case OT problem set needed: flood, conquest, sacrifice, exile, law, covenant, prophets.
- Avoid making triage a blanket moral excuse for every hard text.

### Phase 9 — Need For External Mediator

The OT diagnostic chain ends by deriving the shape of the needed solution:

```text
External(G) ∧ JusticePreserved ∧ MercyEnacted ∧ PersonTransformed ∧ DeathBoundaryAddressed
```

This is the correct handoff to the NT paper.

Gap to fill before Lean:

- Keep this as required profile, not yet historical identification.
- Historical Jesus/resurrection comes after profile derivation.
