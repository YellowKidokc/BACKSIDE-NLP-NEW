# Build-Then-Break Protocol — Christianity Unique Convergence

Run context: 2026-05-18  
Purpose: build the paper up fairly, then destroy-test it without confusing scientific mechanism scoring with theological/teleological claims.

## Core Distinction

Ordinary science is strongest at `how`: mechanism, measurement, prediction, error bounds, falsification, repeatability.

Theophysics is attempting a bridge: `how` plus `why`. The Master Equation is not merely being used as a metaphor for salvation; the internal claim is that the same coherence structure expresses physical law, moral law, and theological telos at different registers.

That bridge must be labeled. If a claim is theological, narrative, metaphysical, formal, historical, or scientific, it should be scored in the correct lane.

## Lanes

| Lane | Can 7Q Score? | Needs |
|---|---:|---|
| Scientific/mechanistic | Yes | mechanism, prediction, boundary, error case |
| Formal/mathematical | Yes | definitions, theorem, dependencies, countermodel |
| Historical | Partly | source criteria, alternatives, uncertainty |
| Theological | Partly | Scripture anchors, orthodox boundaries, interpretive caution |
| Metaphysical | Partly | premise map, rival models, hidden assumptions |
| Narrative/rhetorical | No, not as proof | purpose label, refrain, non-proof status |

## Build Pass

For each claim:

1. Add the strongest in-framework support.
2. Add Scripture only where the interpretation is defensible.
3. Add formal/Lean support only where the theorem actually bears weight.
4. Add historical support without overstating certainty.
5. Preserve recurring motifs as refrains, not as fake proofs.

## Break Pass

For each claim:

1. Ask what a serious theologian would reject.
2. Ask what a serious physicist would call category error.
3. Ask what a historian would say remains underdetermined.
4. Ask what the 7Q station is allowed to score.
5. Eject or weaken any claim that cannot survive its own lane.

## Immediate Next Action

Do not rewrite the paper yet. First build the appendix ledger, run independent build/break passes, then restructure the paper around the surviving chain.
