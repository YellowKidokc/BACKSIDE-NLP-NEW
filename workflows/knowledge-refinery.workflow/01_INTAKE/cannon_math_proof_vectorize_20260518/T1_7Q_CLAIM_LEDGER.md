# T1 7Q Claim Ledger

Purpose: station-facing version of the T1 spine theorem, written as explicit claims with mechanisms, boundaries, and kill conditions.

## Claim 1 — Closure Theorem

Claim: A corrupted system cannot reconstruct its own uncorrupted reference state using only operations closed over the corrupted class.

Mechanism: If every self-generated operation maps error states to error states, then any finite composition of those operations also maps error states to error states.

Boundary: This proves closure under definitions; it does not prove Christianity, personal God, or product-form coherence.

Kill condition: Exhibit a self-generated operation that is closed over the corrupted class but produces a reference-state output from an error-state input.

## Claim 2 — Reference Priority

Claim: If an uncorrupted reference cannot be constructed from corrupted states alone, then the reference must be prior to or independent of the corruption.

Mechanism: A standard not reachable from the error class cannot be generated by the error class.

Boundary: This supports truth-before-error as formal dependency, not yet moral goodness or personal divinity.

Kill condition: Show that an error-only state space can generate its own normatively binding uncorrupted reference without importing a reference from outside the error class.

## Claim 3 — Parasitic Corruption

Claim: Corruption is derivative because it depends on a reference state it did not create.

Mechanism: Deviation requires a reference; distortion requires something to distort.

Boundary: This does not mean corruption is weak or unreal; it means corruption is not foundational.

Kill condition: Provide a coherent model where corruption is primitive and still normatively identifiable as corruption without borrowing a reference standard.

## Claim 4 — Self-Repair Impossibility

Claim: If corrupted operations are closed over the corrupted class, self-generated repair cannot restore the uncorrupted reference.

Mechanism: Finite compositions of class-preserving operations preserve class membership.

Boundary: Repentance may set openness greater than zero, but openness is not itself the external restoring operation.

Kill condition: Show a corrupted system reversing its reference-class status through only internally generated operations.

## Claim 5 — External Grace Requirement

Claim: If self-repair cannot access the uncorrupted reference, restoration requires a non-self-generated operation that carries or accesses the reference.

Mechanism: A reference-state output requires an operation not closed over the error class.

Boundary: This proves need for external reference-bearing input under the model; it does not yet prove the input is personal, incarnate, or Christian.

Kill condition: Demonstrate restoration to the uncorrupted reference without either self-repair or external reference-bearing input.

## Claim 6 — Incarnation Direction

Claim: If the external reference-bearing operation restores rather than replaces the corrupted system, it must be compatible with the system it restores.

Mechanism: Restoration requires interaction with the existing system while preserving continuity.

Boundary: This narrows toward mediator/incarnation logic but does not prove incarnation uniqueness.

Kill condition: Provide a non-compatible external operator that restores the same corrupted system without replacement, coercion, or loss of continuity.

## Claim 7 — Product Form Distinction

Claim: T1 gives the direction of restoration, while product coherence gives the severity of collapse; they are related but distinct.

Mechanism: T1 is a closure theorem about access to reference; product form is a necessary-condition theorem about integrated coherence.

Boundary: T1 can survive even if product form needs repair.

Kill condition: Show that T1 secretly depends on multiplicative product coherence rather than closure over error states.
