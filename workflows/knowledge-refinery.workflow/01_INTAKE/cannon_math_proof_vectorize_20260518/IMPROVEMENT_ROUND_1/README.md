# IMPROVEMENT_ROUND_1

Destination folder for the *changes* that flow out of `..\QUESTION_CHAIN_REVIEW_ROUND\`. Empty at session-start; gets populated as David decides which objections to accept and what changes follow.

## What goes here

- **Revised Q-files** — e.g., `Q04_resurrection_uniqueness_v2.md` with the rewritten hook
- **New Q-files** — e.g., `Q14_5_felt_fine_objection.md` for the affective question if accepted
- **Merged Q-files** — e.g., `Q08_09_merged_why_beings_who_can_refuse.md`
- **Per-format extracts** — `Q##_family.md`, `Q##_thread.md`, `Q##_substack.md`
- **Decision log** — `DECISIONS.md` recording what was accepted/rejected/deferred from the review round and why
- **Cross-reference patch** — script or manual patch adding "Comes from / Leads to" headers to every Q-file
- **Index revision** — updated `QUESTION_CHAIN_INDEX_v2.md` reflecting the new numbering after cuts/merges/adds

## What does NOT go here

- Objections themselves — those live in `..\QUESTION_CHAIN_REVIEW_ROUND\`
- Framework-level changes (Lean modules, axiom additions) — those go to the kernel and to `..\` not here
- Per-AI lane drafts on layers other than what was changed — only files that actually changed should land here

## Promotion path

When IMPROVEMENT_ROUND_1 is stable and reviewed, the revised files get promoted out of this folder and into the main `..\` directory, replacing the originals (originals archived). This folder then either gets archived or stays as the historical record of round 1; round 2 starts in a new folder.

## Convention

Versioned filename suffixes: `_v2`, `_v3`. Once a file is promoted out, the version suffix is dropped.
